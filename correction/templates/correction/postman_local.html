Problème d'Accès à Localhost avec Postman

Résoudre l'erreur d'agent Postman pour les API locales

Le Problème

Vous essayez de tester une API locale (localhost) avec Postman, mais vous rencontrez ce message d'erreur :

"Lors du test d'une API localement, vous devez utiliser l'agent de bureau Postman. Vous avez actuellement un agent différent sélectionné, qui ne peut pas envoyer de demandes à Localhost."

Pourquoi cela arrive-t-il ?

Postman propose deux modes pour envoyer des requêtes :

  • L'agent de bureau : Utilise l'application Postman installée sur votre ordinateur
  • L'agent cloud : Envoie les requêtes via les serveurs de Postman (par défaut dans certaines versions)

Les requêtes vers localhost ou 127.0.0.1 ne peuvent pas être traitées par l'agent cloud car celui-ci ne peut pas accéder à votre machine locale.

Postman Interface
GET http://localhost:3000/api/users Send
✓ Agent de bureau connecté
Les requêtes locales seront envoyées via l'application Postman

Interface Postman avec l'agent de bureau activé

Solutions

Voici comment résoudre ce problème :

1 Téléchargez et installez l'application Postman Desktop

Si vous utilisez la version web de Postman, vous devez installer l'application de bureau.

Rendez-vous sur https://www.postman.com/downloads/ et téléchargez la version pour votre système d'exploitation.

2 Ouvrez Postman Desktop

Lancez l'application Postman que vous venez d'installer. Elle peut se synchroniser avec votre compte Postman existant.

3 Changez l'agent dans Postman Web

Si vous préférez utiliser Postman dans votre navigateur :

  1. Ouvrez Postman Web
  2. Cliquez sur l'icône d'agent en bas à droite de la fenêtre
  3. Sélectionnez "Agent de bureau"
  4. Assurez-vous que l'application Postman Desktop est ouverte
4 Utilisez votre IP locale au lieu de localhost

Une alternative est d'utiliser votre adresse IP locale au lieu de localhost :

# Au lieu de : http://localhost:3000/api # Utilisez (sur Windows) : http://127.0.0.1:3000/api # Ou trouvez votre IP locale avec ipconfig (Windows) ou ifconfig (Mac/Linux) # et utilisez-la : http://192.168.1.XX:3000/api
Conseil supplémentaire : Assurez-vous que votre serveur local est bien démarré et écoute sur le port que vous essayez d'interroger avec Postman.

Vérification

Après avoir appliqué l'une de ces solutions, votre requête vers localhost devrait fonctionner correctement. Vous verrez un indicateur en bas à droite de Postman confirmant que l'agent de bureau est utilisé.
