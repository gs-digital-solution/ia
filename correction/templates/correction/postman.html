Guide de Test d'API avec Postman

Comment configurer une requête API avec seulement une URL et du JSON

Étapes à Suivre

  1. Ouvrez Postman et créez une nouvelle requête
  2. Sélectionnez la méthode HTTP (GET, POST, etc.) selon les besoins de votre API
  3. Collez l'URL de l'API dans le champ prévu à cet effet
  4. Cliquez sur l'onglet "Body" (s'il n'est pas visible, cherchez dans les options)
  5. Sélectionnez "raw" et choisissez "JSON" dans le menu déroulant
  6. Collez votre JSON dans la zone de texte
  7. Ajoutez les en-têtes nécessaires dans l'onglet "Headers" (Content-Type: application/json est souvent requis)
  8. Cliquez sur "Send" pour exécuter la requête

Conseils Importants

  • Si vous ne voyez pas l'onglet "Body", cherchez un bouton pour afficher plus d'options
  • Pour les requêtes POST, PUT, PATCH, le corps (body) est généralement requis
  • Pour les requêtes GET, vous utiliserez généralement l'onglet "Params" plutôt que "Body"
  • Vérifiez la documentation de l'API pour connaître les exigences spécifiques
Note : Vous avez raison, les points 4 et 5 de la configuration (Headers et Authorization) ne sont pas toujours obligatoires. Beaucoup d'APIs fonctionnent avec seulement l'URL et le JSON correctement formaté.

Exemple de JSON pour le Body

{
  "username": "votre_nom_utilisateur",
  "email": "votre@email.com",
  "password": "votre_mot_de_passe"
}

Où trouver l'onglet Body?

Si vous ne voyez pas l'onglet Body, il se peut qu'il soit caché dans un menu déroulant ou que votre fenêtre Postman ne soit pas assez large. Essayez de redimensionner la fenêtre ou de chercher un bouton "▼" qui pourrait afficher plus d'options.

Résultat attendu

Après avoir cliqué sur "Send", vous devriez voir la réponse de l'API dans la partie inférieure de la fenêtre Postman, avec le code de statut, le corps de la réponse et les en-têtes.
